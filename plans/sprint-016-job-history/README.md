# Sprint 016 - Job History

## Goal

Replace the placeholder Jobs route with a real job history workspace.

## Scope

- Load recent jobs from the existing API.
- Search jobs by ID, URL, platform, and stage.
- Filter by job status.
- Sort newest/oldest/progress.
- Open any job detail from the list.

## Done Criteria

- `/jobs` shows a usable job history page.
- Empty, filtered-empty, loading, and error states are handled.
- Frontend typecheck and build pass.
